# Бизнесовые требования

source_id: business_requirements

Улучшенная классификация должна помогать пользователю, а не просто выглядеть умной.

Требования:

1. Уменьшить количество ручных исправлений.
2. Объяснять простыми словами, почему категория выбрана.
3. Учитывать личные правила пользователя.
4. Не менять прошлые транзакции молча.
5. Показывать спорные случаи отдельно.
6. Не раскрывать лишние детали транзакций в отчётах.
7. Работать даже без полного состава покупки.
8. Не делать слишком много мелких категорий, чтобы приложение не стало сложным.

Для пилота достаточно проверить 5–7 понятных правил:

- супермаркет + товары для питомца → Питомец;
- супермаркет + товары для гостей → Встречи и гости;
- супермаркет + бытовые товары → Дом и быт;
- аптека + средства ухода → Уход за собой;
- маркетплейс + книга или курс → Обучение;
- маркетплейс + подарок → Подарки;
- рабочий обед → Обед на работе.
