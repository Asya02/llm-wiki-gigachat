# Smart Spending Wiki — demo raw pack

Это синтетический набор сырых источников для демонстрации LLM-Wiki.

Сюжет: банковское приложение хочет улучшить классификацию расходов. Сейчас оно часто относит покупку к категории по магазину: супермаркет → «Еда», аптека → «Здоровье», маркетплейс → «Покупки». Но для пользователя смысл траты может быть другим: корм для питомца из супермаркета — это «Питомец», торт и свечи — «Встречи и гости», книга — «Обучение», шампунь из аптеки — «Уход за собой».

Цель демо: агент должен построить wiki, где есть категории, правила, исключения, спорные случаи, объяснения переклассификаций и итоговый отчёт для продуктовой команды банка.

## Как использовать

1. Распакуйте папку.
2. Дайте агенту файл `prompts/demo_prompt_full.md`.
3. Агент должен читать `raw/`, не изменять `raw/`, создать `wiki/`, `reports/`, `exports/`.
4. Хороший результат можно открыть как обычную markdown-wiki.

## Что должно получиться

Главные файлы результата:

- `wiki/index.md`
- `wiki/log.md`
- `wiki/overview.md`
- `wiki/categories/*.md`
- `wiki/rules/*.md`
- `wiki/transactions/reclassified-transactions.md`
- `wiki/transactions/needs-review.md`
- `wiki/contradictions/merchant-first-vs-purpose-first.md`
- `reports/final_recommendation.md`
- `exports/recategorized_transactions.csv`
- `exports/category_rules.json`
- `exports/llms.txt`

## Главное, что должна показать демка

Не все расходы нужно классифицировать по типу магазина. Лучше учитывать цель покупки, заметки пользователя, ручные исправления и уверенность в классификации.
